# Run quality gate before Integration Pull Request

The orchestrator invokes a configurable per-repo Quality Gate after a successful Agent Run and before creating an Integration Pull Request. The gate runs as an external CLI (not an importable Python dependency) so that Rangkai does not couple to Saringan's internals. If the gate returns a non-passed status — `failed`, `error`, timeout, or invalid output — the implementation branch and worktree are preserved for inspection, the implementation issue is labeled `needs-triage`, and no Integration Pull Request is created. If the gate passes, the orchestrator proceeds to create the Integration Pull Request, after which remote CI runs as usual. The gate is disabled by default to preserve existing behavior for repos without Saringan.
